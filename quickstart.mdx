## Overview

This guide will walk you through the essential features of HiBeat Pro, from launching the app to playing your first track. By the end, you'll understand how to search for music, stream tracks, download for offline playback, and manage your listening experience.

<Note>
Make sure you've completed the [installation](/installation) before proceeding with this guide.
</Note>

## First Launch

<Steps>
  <Step title="Start the Application">
    Launch HiBeat Pro on your Android device. The app performs an automatic setup:
    
    ```typescript
    // App.tsx initialization
    useEffect(() => {
      SplashScreen.hide();
      SetupPlayer();
    }, []);
    ```
    
    This initializes the **Track Player service** with background playback capabilities.
  </Step>

  <Step title="Network Connection Check">
    HiBeat Pro automatically detects your internet connection:
    
    ```typescript
    useEffect(() => {
      const unsubscribe = NetInfo.addEventListener(state => {
        setIsConnected(state.isConnected);
      });
      return () => unsubscribe();
    }, []);
    ```
    
    <Warning>
    An active internet connection is required for streaming and searching. Downloaded tracks can be played offline.
    </Warning>
  </Step>

  <Step title="Explore the Home Screen">
    The home screen displays:
    - **Top Artists** - Popular and trending artists
    - **Album Releases** - Latest album drops
    - **Random Playlists** - Curated playlist recommendations
    - **Active Track Card** - Currently playing track (bottom)
  </Step>
</Steps>

## Searching for Music

<Tabs>
  <Tab title="Search Tracks">
    ### Find Songs by Name
    
    1. Navigate to the **Search** tab
    2. Enter a song name, artist, or keyword
    3. Browse results with real-time search
    
    The search uses YouTube Music API:
    
    ```typescript
    // hooks/UseSearch/UseSearch.ts
    const ytmusic = new YTMusic();
    
    const searchFunction = async (query: string) => {
      await ytmusic.initialize();
      const results = await ytmusic.search(query);
      return results;
    };
    ```
    
    <Tip>
    Search results include tracks, artists, albums, and playlists - all in one unified interface.
    </Tip>
  </Tab>
  
  <Tab title="Browse Artists">
    ### Explore Artist Profiles
    
    1. Tap any artist card from search or home screen
    2. View their full discography
    3. See top tracks and albums
    
    Artist data is fetched dynamically:
    
    ```typescript
    // hooks/UseYtMusic/UserYtMusic.ts
    const artistInfoFunction = async (artistId: string) => {
      await ytmusic.initialize();
      const artistInfo = await ytmusic.getArtist(artistId);
      return artistInfo;
    };
    ```
  </Tab>
  
  <Tab title="Browse Albums">
    ### Discover Full Albums
    
    1. Search for an album or browse releases
    2. Tap to view the complete tracklist
    3. Play individual tracks or the entire album
    
    Albums include full metadata and artwork extracted from YouTube Music.
  </Tab>
</Tabs>

## Playing Music

### Stream a Track

<Steps>
  <Step title="Select a Track">
    Tap any song from search results, albums, or playlists.
  </Step>

  <Step title="Automatic Streaming Setup">
    HiBeat Pro fetches the streaming URL and loads the track:
    
    ```typescript
    // services/streaming/StreamingTrack.ts
    export const getStreamingData = async (idTrack: string) => {
      const url = `https://www.youtube.com/youtubei/v1/player?key=${keyId}`;
      
      const requestBody = {
        contentCheckOk: true,
        context: {
          client: {
            clientName: 'IOS_MUSIC',
            clientVersion: '5.26.1'
          }
        },
        videoId: idTrack
      };
      
      const response = await axios.post(url, requestBody);
      const format = response.data.streamingData.adaptiveFormats[7];
      
      return {
        id: response.data.videoDetails.videoId,
        url: format.url,
        title: response.data.videoDetails.title,
        artist: response.data.videoDetails.author,
        artwork: response.data.videoDetails.thumbnail.thumbnails[4].url,
        duration: response.data.videoDetails.lengthSeconds
      };
    };
    ```
  </Step>

  <Step title="Playback Controls">
    Use the player interface:
    - **Play/Pause** - Control playback
    - **Skip Forward/Back** - Navigate between tracks
    - **Seek Bar** - Jump to any position
    - **Queue Management** - View and reorder upcoming tracks
  </Step>
</Steps>

### Background Playback

<Info>
HiBeat Pro supports background playback with full notification controls.
</Info>

The Track Player is configured for background operation:

```typescript
// services/TrackPlayerService/SetupPlayer.ts
export const SetupPlayer = async () => {
  const options = {
    autoHandleInterruptions: true,
    permanent: true,
  };
  
  await TrackPlayer.setupPlayer(options);
  await TrackPlayer.updateOptions({
    android: {
      appKilledPlaybackBehavior:
        AppKilledPlaybackBehavior.StopPlaybackAndRemoveNotification,
    },
    capabilities: [
      Capability.Play,
      Capability.Pause,
      Capability.SkipToNext,
      Capability.SkipToPrevious,
      Capability.Stop,
      Capability.Skip,
    ],
    progressUpdateEventInterval: 2,
  });
};
```

**Features:**
- Continue listening when screen is locked
- Full notification controls
- Automatic handling of interruptions (calls, alarms)
- Persists across app switching

## Downloading Music

<Warning>
Download features are **Android-only** and require storage permissions.
</Warning>

<Steps>
  <Step title="Enable Permissions">
    On first download, grant storage permissions when prompted.
  </Step>

  <Step title="Download a Track">
    1. Tap the **menu icon** (three dots) on any track
    2. Select **Download**
    3. Monitor progress in notification area
    
    The download process:
    
    ```typescript
    // services/downloader/Downloader.ts
    const downloadFile = async (format: string, trackInfo: Song, trackImg: string) => {
      const {fs} = RNFetchBlob;
      const safeName = trackInfo.name.replace(/[^a-zA-Z0-9]/g, '_');
      const destinationPath = `${fs.dirs.DownloadDir}/${safeName}.m4a`;
      
      // Download audio file
      await RNFetchBlob.config({
        addAndroidDownloads: {
          useDownloadManager: true,
          mime: 'audio/aac',
          notification: true,
          path: destinationPath,
          mediaScannable: true
        }
      }).fetch('GET', format);
      
      // Download cover art
      await RNFetchBlob.config({
        fileCache: true, 
        path: tempImagePath
      }).fetch('GET', trackImg);
      
      // Embed metadata with FFmpeg
      const ffmpegCommand = `-y -i "${destinationPath}" -i "${tempImagePath}" 
        -metadata Title="${trackInfo.name}" 
        -metadata Artist="${trackInfo.artist.name}" 
        -metadata Album="${trackInfo.album.name}" 
        -map 0 -map 1 -c copy -disposition:v:0 attached_pic "${tempPath}"`;
      
      await FFmpegKit.execute(ffmpegCommand);
    };
    ```
  </Step>

  <Step title="Access Downloaded Files">
    Downloaded tracks are saved to:
    ```
    /storage/emulated/0/Download/
    ```
    
    Files include:
    - High-quality audio (M4A format)
    - Embedded album artwork
    - Full metadata (title, artist, album)
  </Step>
</Steps>

<Check>
**Success Notification:** You'll see a toast message when download completes successfully.
</Check>

## Key Features Walkthrough

### 1. State Management with Zustand

HiBeat Pro uses Zustand for lightweight, efficient state management:

```typescript
// store/trackStackStore/GetTrackStore.ts
import {create} from 'zustand';

export const useTrackStore = create((set) => ({
  currentTrack: null,
  setCurrentTrack: (track) => set({currentTrack: track}),
  queue: [],
  addToQueue: (track) => set((state) => ({
    queue: [...state.queue, track]
  }))
}));
```

### 2. Data Fetching with TanStack Query

Efficient API calls with automatic caching:

```typescript
// hooks/UseYtMusic/UserYtMusic.ts
export const useArtistInfoById = (artistId: string) => {
  return useQuery({
    queryKey: ['artistInfoById', artistId],
    queryFn: () => artistInfoFunction(artistId),
  });
};
```

### 3. Optimized Lists with FlashList

High-performance scrolling for large music libraries:

```typescript
import {FlashList} from '@shopify/flash-list';

<FlashList
  data={tracks}
  renderItem={({item}) => <TrackItem track={item} />}
  estimatedItemSize={72}
/>
```

## Tips for Best Experience

<AccordionGroup>
  <Accordion title="Audio Quality" icon="waveform">
    HiBeat Pro automatically selects the highest quality audio format available (typically 128kbps AAC). For downloads, audio is saved in M4A format with full quality preservation.
  </Accordion>

  <Accordion title="Battery Optimization" icon="battery-full">
    Background playback is optimized for battery life. If experiencing issues:
    - Disable battery optimization for HiBeat Pro in system settings
    - Ensure "Background data" is enabled for the app
  </Accordion>

  <Accordion title="Storage Management" icon="hard-drive">
    Monitor download storage usage:
    - Average track: 3-5 MB
    - Album art embedded in each file
    - Clear app cache periodically in Settings
  </Accordion>

  <Accordion title="Network Usage" icon="signal">
    Streaming data usage:
    - ~1 MB per minute of playback
    - Use WiFi for downloads
    - Downloaded tracks use zero data
  </Accordion>
</AccordionGroup>
