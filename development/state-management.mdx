## Overview

HiBeat Pro uses [Zustand](https://github.com/pmndrs/zustand) for state management. Zustand provides a minimal, hook-based API that's perfect for React Native applications.

## Why Zustand?

<CardGroup cols={2}>
  <Card title="Simple API" icon="code">
    No boilerplate, no providers, just hooks
  </Card>
  
  <Card title="TypeScript First" icon="shield-check">
    Full TypeScript support with type inference
  </Card>
  
  <Card title="Small Bundle" icon="feather">
    Minimal size impact on your app
  </Card>
  
  <Card title="No Context" icon="rocket">
    No need for Context providers or HOCs
  </Card>
</CardGroup>

## Store Architecture

All stores are located in `src/store/` and follow a consistent pattern:

```
store/
├── albumStore/           # Album selection and data
├── playlistStore/        # Playlist management
├── searchStore/          # Search results and query
├── artistStore/          # Artist data
├── trackStackStore/      # Track queue management
├── modalStore/           # Bottom sheet modal state
├── albumResaleStore/     # Album releases
├── randomPlatlistStore/  # Random playlists
└── sheetModalTrack/      # Track modal state
```

## Core Stores

<Tabs>
  <Tab title="Album Store">
    Manages album selection and related data:

    ```typescript src/store/albumStore/albumStore.ts
    import { AlbumSearch } from 'interfaces/AlbumSearch/AlbumSearch';
    import { AlbumRelaseInterface } from 'interfaces/AlbumsRelase/AlbumsRelase';
    import { AlbumSummary } from 'interfaces/ArtistInterface/YTMuiscArtistInterface';
    import {create} from 'zustand';

    interface AlbumStore {
      albumId: string;
      setAlbumId: (albumId: string) => void;

      albumInfoSelected: AlbumSearch | AlbumSummary | null;
      setAlbumsInfoSelected: (album: AlbumSearch | AlbumSummary) => void;

      albumListStore: AlbumRelaseInterface | null;
      setAlbumListStore: (album: AlbumRelaseInterface) => void;
    }

    export const useAlbumStore = create<AlbumStore>(set => ({
      albumId: '',
      setAlbumId: (albumId: string): void => {
        return set(() => ({
          albumId: albumId,
        }));
      },

      albumInfoSelected: null,
      setAlbumsInfoSelected: (album: AlbumSearch | AlbumSummary): void => {
        return set(() => ({
          albumInfoSelected: album,
        }));
      },

      albumListStore: null,
      setAlbumListStore: (album: AlbumRelaseInterface): void => {
        return set(() => ({
          albumListStore: album,
        }));
      },
    }));
    ```

    **Key Features:**
    - Stores current album ID
    - Maintains selected album information
    - Caches album list data
  </Tab>

## Usage Patterns

### Reading State

Access store state in any component:

```typescript
import { useAlbumStore } from 'store/albumStore/albumStore';

function AlbumComponent() {
  const albumId = useAlbumStore(state => state.albumId);
  const albumInfo = useAlbumStore(state => state.albumInfoSelected);
  
  return (
    <View>
      <Text>Album ID: {albumId}</Text>
      <Text>Album: {albumInfo?.name}</Text>
    </View>
  );
}
```

### Updating State

Call setter functions to update state:

```typescript
import { useAlbumStore } from 'store/albumStore/albumStore';

function AlbumSelector() {
  const setAlbumId = useAlbumStore(state => state.setAlbumId);
  const setAlbumsInfoSelected = useAlbumStore(state => state.setAlbumsInfoSelected);
  
  const handleSelectAlbum = (album: AlbumSearch) => {
    setAlbumId(album.browseId);
    setAlbumsInfoSelected(album);
  };
  
  return (
    <TouchableOpacity onPress={() => handleSelectAlbum(album)}>
      <Text>{album.name}</Text>
    </TouchableOpacity>
  );
}
```

### Selecting Multiple Values

```typescript
const { albumId, albumInfoSelected, setAlbumId } = useAlbumStore(
  state => ({
    albumId: state.albumId,
    albumInfoSelected: state.albumInfoSelected,
    setAlbumId: state.setAlbumId,
  })
);
```

### Accessing Store Outside Components

Zustand stores can be accessed outside React components:

```typescript
import { useAlbumStore } from 'store/albumStore/albumStore';

// Get current state
const currentAlbumId = useAlbumStore.getState().albumId;

// Update state
useAlbumStore.getState().setAlbumId('new-album-id');

// Subscribe to changes
const unsubscribe = useAlbumStore.subscribe(
  state => console.log('Album changed:', state.albumId)
);
```

## Store Patterns

<Accordion title="State + Setter Pattern">
  Each piece of state has a corresponding setter function:

  ```typescript
  interface Store {
    value: string;              // State
    setValue: (v: string) => void;  // Setter
  }
  ```

  This pattern:
  - Makes updates explicit and traceable
  - Provides clear API for consumers
  - Supports TypeScript inference
</Accordion>

<Accordion title="Null Initial State">
  Complex data starts as `null` until loaded:

  ```typescript
  interface Store {
    albumInfoSelected: AlbumSearch | null;
    setAlbumsInfoSelected: (album: AlbumSearch) => void;
  }
  ```

  This pattern:
  - Distinguishes between "not loaded" and "loaded but empty"
  - Prevents errors from accessing undefined properties
  - Makes loading states explicit
</Accordion>

<Accordion title="Immutable Updates">
  State updates return new objects:

  ```typescript
  setAlbumId: (albumId: string): void => {
    return set(() => ({
      albumId: albumId,
    }));
  }
  ```

  This pattern:
  - Ensures proper React re-renders
  - Maintains predictable state updates
  - Follows React best practices
</Accordion>

## Best Practices

<CardGroup cols={2}>
  <Card title="Selective Subscriptions" icon="filter">
    Only select the state you need to prevent unnecessary re-renders:
    ```typescript
    const albumId = useAlbumStore(state => state.albumId);
    ```
  </Card>
  
  <Card title="Avoid Derived State" icon="ban">
    Compute derived values in components, not stores:
    ```typescript
    const album = useAlbumStore(state => state.albumInfoSelected);
    const albumName = album?.name || 'Unknown';
    ```
  </Card>
  
  <Card title="Type Safety" icon="shield">
    Always define interfaces for store state:
    ```typescript
    interface AlbumStore {
      albumId: string;
      setAlbumId: (id: string) => void;
    }
    ```
  </Card>
  
  <Card title="Single Responsibility" icon="star">
    Each store manages one domain of state. Don't mix unrelated concerns.
  </Card>
</CardGroup>

## Performance Optimization

### Memoize Selectors

```typescript
import { useMemo } from 'react';

const selectAlbumData = (state) => ({
  id: state.albumId,
  info: state.albumInfoSelected,
});

function AlbumComponent() {
  const albumData = useAlbumStore(selectAlbumData);
  // Component only re-renders when albumId or albumInfoSelected changes
}
```

### Shallow Equality

For multiple values, use shallow equality:

```typescript
import shallow from 'zustand/shallow';

const { albumId, setAlbumId } = useAlbumStore(
  state => ({ albumId: state.albumId, setAlbumId: state.setAlbumId }),
  shallow
);
```

## Testing Stores

```typescript
import { useAlbumStore } from 'store/albumStore/albumStore';

    setAlbumId('test-album-123');
    
    expect(useAlbumStore.getState().albumId).toBe('test-album-123');
  });
});
```
