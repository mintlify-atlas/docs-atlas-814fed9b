## Overview

HiBeat Pro uses **React Native Track Player** to provide seamless background audio playback. Music continues playing even when the app is in the background or the screen is locked.

## Track Player Setup

The Track Player is initialized during app startup with specific configurations for background playback.

### Player Initialization

```typescript
export const SetupPlayer = async () => {
  try {
    const options = {
      autoHandleInterruptions: true,
      permanent: true,
    };
    
    await TrackPlayer.setupPlayer(options);
    await TrackPlayer.updateOptions({
      android: {
        appKilledPlaybackBehavior:
          AppKilledPlaybackBehavior.StopPlaybackAndRemoveNotification,
      },
      capabilities: [
        Capability.Play,
        Capability.Pause,
        Capability.SkipToNext,
        Capability.SkipToPrevious,
        Capability.Stop,
        Capability.Skip,
      ],
      compactCapabilities: [
        Capability.Play,
        Capability.Pause,
        Capability.SkipToNext,
        Capability.SkipToPrevious,
        Capability.Stop,
        Capability.Skip,
      ],
      progressUpdateEventInterval: 2,
    });
  } catch (error) {
    console.log(error);
  }
};
```

**Source:** `src/services/TrackPlayerService/SetupPlayer.ts`

<Note>
The `permanent: true` option enables the player to persist across app restarts.
</Note>

## Remote Controls

The app implements a playback service that responds to remote control events from system media controls, Bluetooth devices, and headphones.

### Playback Service

```typescript
export const PlaybackService = async function () {
  TrackPlayer.addEventListener(Event.RemotePlay, () => TrackPlayer.play());
  TrackPlayer.addEventListener(Event.RemotePause, () => TrackPlayer.pause());
  TrackPlayer.addEventListener(Event.RemoteStop, () => TrackPlayer.stop());
  TrackPlayer.addEventListener(Event.RemoteNext, () => TrackPlayer.skipToNext());
  TrackPlayer.addEventListener(Event.RemotePrevious, () => TrackPlayer.skipToPrevious());
};
```

**Source:** `src/services/TrackPlayerService/TrackPlayerService.ts`

<Info>
These event listeners enable control from:
- Lock screen media controls
- Notification controls
- Bluetooth devices (car stereos, headphones)
- Android Auto and CarPlay
- Smart watches and wearables
</Info>

### State Tracking Hook

```typescript
const events = [Event.PlaybackState, Event.PlaybackError];

export const useIsTrackPlaying = () => {
  const [isPlaying, setIsPlaying] = useState(false);
  
  useTrackPlayerEvents(events, event => {
    if (event.type === Event.PlaybackState) {
      switch (event.state) {
        case State.Playing:
          setIsPlaying(true);
          break;
        case State.Paused:
        case State.Stopped:
        case State.Buffering:
        default:
          setIsPlaying(false);
          break;
      }
    }
  });
  
  useEffect(() => {
    const getInitialState = async () => {
      const state = await TrackPlayer.getState();
      setIsPlaying(state === State.Playing);
    };
    getInitialState();
  }, []);
  
  return isPlaying;
};
```

**Source:** `src/services/TrackPlayerService/TrackPlayerStates.ts`

## Playback Controls

The app provides several utility functions for controlling playback:

### Toggle Playback

```typescript
export const togglePlayback = async () => {
  const state = await TrackPlayer.getState();
  if (state === State.Playing) {
    await TrackPlayer.pause();
  } else {
    await TrackPlayer.play();
  }
};
```

### Previous Track Handler

```typescript
export const handlerPreviousTrack = async () => {
  const progress = await TrackPlayer.getProgress();
  const state = await TrackPlayer.getState();
  
  if (progress.position > 10) {
    // Restart the song if played more than 10 seconds
    await TrackPlayer.seekTo(0);
  } else {
    // Skip to previous track if less than 10 seconds
    await TrackPlayer.skipToPrevious();
  }
  
  // Ensure playback continues
  if (state !== State.Playing) {
    await TrackPlayer.play();
  }
};
```

<Note>
The previous track button intelligently restarts the current track if more than 10 seconds have been played, mimicking the behavior of popular music apps.
</Note>

### Next Track Handler

```typescript
export const handlerNextTrack = async () => {
  const state = await TrackPlayer.getState();
  
  await TrackPlayer.skipToNext();
  
  // Ensure playback continues
  if (state !== State.Playing) {
    await TrackPlayer.play();
  }
};
```

**Source:** `src/services/TrackPlayerService/TrackPlayerEvents.ts`

```typescript
progressUpdateEventInterval: 2
```

- Accurate seek functionality

## Interruption Handling

The player automatically handles audio interruptions:

```typescript
autoHandleInterruptions: true
```

<Accordion title="Handled Interruptions">
The player automatically responds to:
- **Phone calls:** Pauses music during calls, resumes after
- **Alarms:** Ducks audio or pauses temporarily
- **Voice assistants:** Pauses for Siri, Google Assistant, etc.
- **Navigation apps:** Ducks audio for navigation prompts
- **Other audio apps:** Yields to higher priority audio
</Accordion>

## Android-Specific Behavior

When the app is killed on Android:

```typescript
android: {
  appKilledPlaybackBehavior:
    AppKilledPlaybackBehavior.StopPlaybackAndRemoveNotification,
}
```

<Warning>
Playback stops and the notification is removed when the user force-closes the app. This prevents unwanted battery drain and respects user intent.
</Warning>
