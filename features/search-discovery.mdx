## Overview

HiBeat Pro's search functionality leverages the **ytmusic-api** library to provide comprehensive music discovery. Users can search for tracks, artists, albums, and playlists from YouTube Music's vast catalog.

## Implementation

### Search Service

The search functionality is implemented using the **ytmusic-api** library:

```typescript
import YTMusic from 'ytmusic-api';

const ytmusic = new YTMusic();

export const serachTracksFuntion = async (track: string): Promise<SongDetailed[]> => {
  try {
    await ytmusic.initialize();
    const searchInfo = await ytmusic.searchSongs(track);
    return searchInfo;
  } catch (error) {
    console.log(error);
    return null;
  }
};

export const serachArtistFuntion = async (artist: string): Promise<ArtistDetailed[]> => {
  try {
    await ytmusic.initialize();
    const artistInfo = await ytmusic.searchArtists(artist);
    return artistInfo;
  } catch (error) {
    console.log(error);
    return null;
  }
};
```

**Source:** `src/hooks/UseSearch/UseSearchTracks.ts`

<Note>
The ytmusic-api library requires initialization before each search query.
</Note>

### Album Search

Album search uses a custom implementation with YouTube Music's Web Remix API:

```typescript
export const searchAlbumsByAxios = async (somethingQuery: string): Promise<AlbumSearch[]> => {
  const URL = `https://music.youtube.com/youtubei/v1/search?prettyPrint=false`;
  
  const body = {
    context: {
      client: {
        clientName: 'WEB_REMIX',
        clientVersion: '0.1',
      },
    },
    params: 'EgWKAQIYAWoSEAMQBBAOEAoQCRAVEAUQERAQ',
    query: somethingQuery,
    formData: {
      selectedValues: ['GB'],
    },
  };
  
  const response = await axios.post(URL, body, {
    headers: {
      Accept: 'application/json',
      'Content-Type': 'application/json',
    },
  });
  
  // Parse and format album data
  return formatAlbumResults(response.data);
};
```

**Source:** `src/hooks/UseAlbumSearch/UseAlbumSearch.ts`

## Search UI

The search interface consists of a unified search form with tabbed results:

### Search Form Component

```typescript
export const SearchForm: React.FC = () => {
  const {control, handleSubmit} = useForm<FormData>();
  const {setTrackList, setArtistList, setAlbumsList} = useSearchStore();
  
  const onSubmit = async (data: FormData) => {
    const [tracksInfo, artistInfo, albumSearch] = await Promise.all([
      serachTracksFuntion(data.search),
      serachArtistFuntion(data.search),
      searchAlbumsByAxios(data.search),
    ]);
    
    setTrackList(tracksInfo);
    setArtistList(artistInfo);
    setAlbumsList(albumSearch);
  };
  
  return (
    <TextInput
      returnKeyType="search"
      placeholder="Search something"
      onSubmitEditing={handleSubmit(onSubmit)}
    />
  );
};
```

**Source:** `src/components/SearchComponents/components/SearchForm.tsx`

<Info>
The search form uses **Promise.all** to execute all three search queries in parallel for optimal performance.
</Info>

## State Management

Search results are managed using Zustand store:

```typescript
const {setTrackList, setArtistList, setAlbumsList} = useSearchStore();

// Store search results
setTrackList(tracksInfo);
setArtistList(artistInfo);
setAlbumsList(albumSearch);
```

<Accordion title="Search Store Structure">
The search store maintains three separate arrays:
- **trackList:** Array of track search results
- **artistList:** Array of artist search results
- **albumsList:** Array of album search results
</Accordion>

## Search Query Optimization

<Accordion title="Parallel Queries">
All search types execute simultaneously using `Promise.all`, reducing total search time by up to 66%:

```typescript
const [tracks, artists, albums] = await Promise.all([
  serachTracksFuntion(query),
  serachArtistFuntion(query),
  searchAlbumsByAxios(query),
]);
```
</Accordion>

## Error Handling

The search system includes graceful error handling:

```typescript
try {
  const [tracksInfo, artistInfo, albumSearch] = await Promise.all([...]);
  // Update state with results
} catch (error) {
  console.error('Error fetching data:', error);
  // Return null or empty array
}
```

<Warning>
If a search query fails, the app returns null or an empty array to prevent crashes while allowing other search types to complete successfully.
</Warning>

## Integration with React Query

Search results can also be cached using React Query:

```typescript
export const useSearcTracksResult = (track: string): UseQueryResult<SongDetailed[], Error> => {
  return useQuery({
    queryKey: ['useSearchTracksKey', serachTracksFuntion],
    queryFn: () => serachTracksFuntion(track),
  });
};
```

<Info>
React Query integration enables automatic caching and refetching of search results.
</Info>
